# 新規実験フォルダ作成

KAGGLE_DIRECTION.md の命名規則に従って実験フォルダを作成する。

## 命名規則

- **Claude用**: `workspace/exp(アルファベット)(数字2桁)_(実験名)` 例: `workspace/expA00_baseline`
- **人間用** (`--human` 指定時): `workspace/exp(数字3桁)_(実験名)` 例: `workspace/exp200_try_unet`

## 手順

1. `workspace/` 内の既存フォルダを確認し、次の番号を自動決定する
2. `$ARGUMENTS` から実験名を取得（`--human` フラグがあれば人間用の命名を使う）
3. フォルダを作成し、以下のサブディレクトリも作る:
   - `results/` - 実験結果
   - `dataset/` - データセット
   - `src/` - ソースコード
4. `SESSION_NOTES.md` を以下のテンプレートで作成:

```markdown
# SESSION_NOTES: {フォルダ名}

## セッション情報
- **日付**: {今日の日付}
- **作業フォルダ**: {フォルダパス}
- **目標**: {実験名から推定 or 空欄}

## 仮説
<!-- 何を良くしたいのか -->

## 試したアプローチと結果

| アプローチ | 変更点 | CV | LB | 備考 |
|-----------|--------|-----|-----|------|
| - | - | - | - | - |

## ファイル構成
<!-- 作成したスクリプト、可視化結果、データファイル -->

## 重要な知見
<!-- セッション中の発見、避けるべきアプローチ、有効だったテクニック -->

## 性能変化の記録

| 実験 | 変更内容 | 結果 | 改善幅 |
|------|---------|------|--------|
| - | - | - | - |

## コマンド履歴
```bash
# 再現性のための記録
```

## 次のステップ
- [ ] TODO
```

5. 作成したフォルダのパスを報告する
