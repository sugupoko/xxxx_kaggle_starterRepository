# EDA実行スキル

## 手順

1. **データの特定**: `$ARGUMENTS` でパスが指定されていればそれを使う。なければ `datasets/` 内のデータを探す

2. **基本統計の確認**:
   - ファイル一覧とサイズ
   - データ形状 (行数、列数、次元)
   - データ型の確認
   - 欠損値の割合
   - 基本統計量 (mean, std, min, max, quartiles)

3. **データの詳細分析**:
   - ターゲット変数の分布
   - 特徴量の分布とヒストグラム
   - カテゴリ変数のユニーク数とvalue_counts
   - 相関行列（数値変数間）
   - train/testの分布比較

4. **画像/3Dデータの場合** (該当する場合):
   - サンプル画像の可視化
   - ピクセル値の分布
   - チャネル情報
   - 解像度の確認

5. **結果を `competition/overview.md` に保存**。以下のフォーマットで:

```markdown
# コンペティション概要

## 基本情報
- コンペ名:
- 評価指標:
- 提出形式:
- データサイズ:

## データ構造
<!-- テーブル/画像/3Dなどの構造詳細 -->

## 主要な発見
<!-- EDAで得られた重要な知見 -->

## 注意点
<!-- データの特殊性、リーク可能性、クラス不均衡など -->

## 推奨アプローチ
<!-- データ特性に基づく推奨手法 -->
```

6. 可視化結果は `competition/` に画像として保存する
